# 역할
너는 글로벌 핀테크 스타트업의 수석 데이터 분석가이자,
Python 기반 데이터 시각화 및 Streamlit 관제 대시보드를 구축하는 보안 엔지니어다.

global_fds.db SQLite 데이터베이스를 기반으로,
글로벌 결제 내역을 모니터링하고 이상거래(Fraud) 패턴을 실시간으로 추적·증명할 수 있는
"글로벌 핀테크 실시간 이상거래 탐지(FDS) 시스템 대시보드 프로그램"을 작성한다.

# 프로젝트 구조
프로젝트 루트: ./5.DatabaseSQL/

- data/
  - global_fds.db (SQLite 파일)
- code/
  - (여기에 streamlit 분석 프로그램 생성)

# 데이터베이스 정보
파일: ./5.DatabaseSQL/data/global_fds.db
테이블: global_payments
컬럼 구조:
- transaction_id (text) -> 결제 고유 ID (TXN_1001 형태)
- user_id (text) -> 고객 고유 ID
- timestamp (text) -> 결제 일시 (YYYY-MM-DD HH:MM:SS)
- amount_usd (real) -> 결제 금액 (USD)
- country (text) -> 결제 발생 국가 (US, KR, SG, GB, JP)
- merchant_category (text) -> 가맹점 업종 (Gaming, Luxury, Food, Electronics, Travel)
- device_ip (text) -> 결제 기기 가상 IP
- is_fraud (integer) -> 이상거래 여부 (0: 정상, 1: 위험 탐지)

# 프로그램 목표
Streamlit 기반 "글로벌 핀테크 실시간 FDS 관제 및 리스크 분석 시스템" 구축

이 프로그램은 단순 시각화가 아니라:
- 글로벌 리스크 및 이상거래 발생 현황 탐색 (EDA)
- 정상 거래 대비 이상거래의 금액적 규모 및 피해 차단 액수(ROI) 추적
- 시간대별, 국가별, 업종별 리스크 전파 추세 분석
- 취약점 분석을 통한 FDS 보안 규칙(Rule) 고도화 제안
이 가능한 “핀테크 보안 통합 관제 도구”여야 한다.

# 필수 기능 요구사항

## 1. 데이터베이스 연결 및 최적화
- sqlite3 또는 SQLAlchemy를 사용하여 로컬의 global_fds.db 연결
- @st.cache_data를 활용해 대용량 데이터 로딩 속도를 최적화할 것
- DB 연결 실패 시 사용자가 직관적으로 알 수 있는 경고 메시지(st.error) 표시 및 예외 처리

## 2. 사이드바 실시간 필터 시스템 (Interactive Sidebar)
다음 조건을 Streamlit sidebar에서 실시간으로 선택 및 연동 가능하도록 구성:
- 국가(country) : 멀티 선택 (기본값: 전체 선택)
- 업종(merchant_category) : 멀티 선택 (기본값: 전체 선택)
- 리스크 등급(is_fraud) : 전체, 정상 거래만(0), 이상거래만(1) 선택 가능한 라디오 버튼
- 결제 금액 범위(amount_usd) : 최소값~최대값 슬라이더 (st.slider)
필터 변경 시 대시보드의 모든 KPI 메트릭과 시각화 차트가 즉각 동기화되도록 구현

## 3. 데이터 탐색 및 리스크 요약 영역
- 필터링된 최종 DataFrame 테이블을 화면에 명확히 표시 (st.dataframe)
- 이상거래(is_fraud=1)인 행은 사용자가 쉽게 식별할 수 있도록 데이터프레임 내부 하이라이트 기능 적용
- 현재 필터링 상태 요약(총 거래 건수, 고유 사용자 수, 탐지된 IP 개수)을 상단에 요약 표시

## 4. 핵심 금융 보안 KPI 지표 영역 (Metric Card)
Streamlit의 `st.metric`을 사용하여 상단에 가로로 배치:
- **총 결제 모니터링 규모**: 현재 필터링된 데이터의 총 결제 금액 합계 ($ 표시)
- **FDS 이상거래 탐지 건수**: 탐지된 이상거래(is_fraud=1)의 총 건수 및 전체 거래 대비 발생 비율(%)
- **피해 예방 차단 금액**: 탐지된 이상거래 금액의 총합 (보안 시스템이 지켜낸 자산 가치)
- **최고 위험 업종/국가**: 이상거래 발생 비율이 가장 높은 업종명 혹은 국가명 출력

## 5. 분석 목적별 시각화 섹션 (Plotly 활용 권장)
그래프는 확대, 마우스 오버 툴팁, 범례 제어가 가능해야 한다.

### (1) 일별/시간별 리스크 발생 추세 분석 (Line / Area Chart)
- x축 = 결제 날짜(timestamp에서 파싱한 일자), y축 = 총 결제액 또는 이상거래 건수
- 그룹(Color) = is_fraud 여부 혹은 country
- 목적: 특정 날짜나 시간대(예: 새벽 시간)에 리스크가 급증하는 패턴 분석

### (2) 국가별/업종별 이상거래 금액 비교 분석 (Grouped Bar Chart)
- x축 = country 또는 merchant_category, y축 = 총 결제 금액
- 정상 거래(is_fraud=0)와 이상거래(is_fraud=1)의 규모를 나란히 비교하는 막대그래프
- 목적: 어떤 국가나 업종에서 이상거래가 집중적으로 도용되고 있는지 파악

### (3) 이상거래 결제 금액 분포 (Box Plot / Histogram)
- 정상 거래와 이상거래의 결제 금액(amount_usd) 분포 비교 및 이상치(Outlier) 탐색
- 목적: FDS 차단 룰을 설정하기 위한 적정 '고액 결제 임계값' 산정 기준 제시

### (4) 시간대 및 국가별 리스크 매트릭스 (Heatmap)
- x축 = 결제 시간(00시~23시), y축 = country
- 내부 값 = 이상거래 발생 건수 또는 총 위험 금액
- 목적: 시차를 고려한 글로벌 해킹/도용 세력의 집중 공격 타이밍 포착

### (5) 리스크 기여도 Top-N 위험 사용자 탐색 (Horizontal Bar Chart)
- 이상거래 건수 또는 금액이 가장 높은 위험 User_ID TOP 5 / TOP 10 시각화
- 목적: 블랙리스트 계정 등록 및 긴급 계정 동결 조치를 위한 레퍼런스 데이터 제시

## 6. AI 기반 FDS 보안 관제 자동 요약 리포트 (Dynamic Report)
통계 연산 결과를 조합하여 동적으로 변하는 텍스트 리포트 영역 구현
(예시: "2026년 5월 글로벌 결제 관제 결과, 총 OOO건의 거래 중 OO건의 이상거래가 성공적으로 탐지 및 차단되었습니다. 이를 통해 총 $OOO의 금융 피해를 예방했습니다. 특히, 비-US 지역에서 새벽 1시~4시 사이에 발생한 'Luxury' 및 'Electronics' 카테고리의 고액 결제가 주요 피싱 패턴으로 분석되므로 해당 Rule의 차단 강도를 높일 것을 권고합니다.")
