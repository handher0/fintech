9. 멀티유저/멀티세션/저장기능 챗봇

# 목적
- Supabase를 활용해서 멀티유저/멀티세션/저장이 가능한 RAG 앱을 만듭니다.
- 기존 `multiref.py`를 멀티유저 버전으로 확장합니다.
- **Supabase Authentication은 사용하지 않고**, DB의 `user` 테이블 기반으로 통일합니다.
- 로컬 개발과 **Streamlit Cloud 배포**를 모두 지원하는 코드로 작성합니다.

# 파일명
- `multiusers.py` (메인 앱)
- `multiusers-ref.sql` (DB 스키마)
- `requirements.txt` (의존성)

# 핵심 아키텍처(고정)
- 사용자 관리는 반드시 `user` 테이블에서 직접 관리합니다. (회원가입/로그인 로직을 앱에서 처리)
- `auth.users` 및 Supabase Auth 회원가입/로그인/이메일 인증 흐름은 사용하지 않습니다.
- `멀티세션 ref`에서 만든 `chat_sessions`, `chat_messages` 테이블을 그대로 활용하되, `user_id` 컬럼(FK)을 추가해서 사용자별 데이터 분리를 보장합니다.
- `vector_documents` 테이블에도 `user_id` 컬럼(FK)을 추가합니다.
- 모든 조회/저장/삭제는 항상 `user_id` 조건으로 필터링합니다.

# 화면 흐름 (고정)
- 앱 실행 시 **로그인/회원가입 전용 화면**이 먼저 표시됩니다.
- 미로그인 상태에서는 채팅, 세션 관리, 사이드바 대시보드 UI를 표시하지 않습니다.
- **로그인 성공** 또는 **회원가입 성공** 시 메인 대시보드(헤더 + 사이드바 + 채팅)로 이동합니다.
- 회원가입 성공 시 별도 로그인 없이 **자동 로그인** 후 대시보드로 이동합니다.
- 사이드바 **로그아웃** 버튼 클릭 시 로그인/회원가입 화면으로 돌아갑니다.

# 상세 기능
- Supabase 기반 사용자 로그인/회원가입 기능 추가 (`login_id`, `password`)
  - 비밀번호는 평문 저장 금지, PBKDF2-SHA256 해시 저장 필수
  - 로그인 성공 시 해당 사용자 기준으로만 세션/메시지/벡터 데이터 접근
- Supabase 기반 세션 저장/로드/삭제 기능 추가
- 세션 관리 버튼/동작은 `multiref.py`와 동일한 UX를 유지
  - 세션저장 / 세션로드 / 세션삭제 / 화면초기화 / vectordb
  - 세션 선택 시 자동 로드
  - 대화·PDF 처리 후 자동 저장
  - LLM 기반 세션 제목 자동 생성

## 화면 UI 및 답변 스타일
- UI와 답변 스타일(display, color 등)은 `ref.py`, `multiref.py`를 참고합니다.
- 챗봇 이름: `"숭실대학교 RAG 챗봇"`

### 로그인/회원가입 화면
- 사이드바가 아닌 **화면 중앙**에 로그인·회원가입 UI를 배치합니다.
- 로고, 제목, 로그인/회원가입 탭을 표시합니다.
- `ref.py`와 동일한 색상 스타일(핑크/골드/블루 헤딩, 핑크 버튼)을 적용합니다.

### 메인 대시보드 (로그인 후)
- `ref.py` 스타일 헤더(로고 + 제목) 표시
- 사이드바: 로그아웃, 세션 관리, PDF 업로드, 모델 정보
- 본문: 환영 메시지 + 채팅 UI (스트리밍 답변, 후속 질문 3개)

# DB 스키마 변경 요구사항 (중요)
- SQL 파일: `multiusers-ref.sql` (Supabase SQL Editor에서 처음부터 실행)
- `user` 테이블 생성 (`login_id`, `password_hash`)
- 아래 테이블에 `user_id` 컬럼을 추가/반영합니다.
  - `chat_sessions.user_id` (NOT NULL, `user.id` FK)
  - `chat_messages.user_id` (NOT NULL, `user.id` FK)
  - `vector_documents.user_id` (NOT NULL, `user.id` FK)
  - 필요 시 인덱스 추가: `(user_id)`, `(user_id, session_id)`
- `match_vector_documents` RPC에 `filter_user_id` 파라미터 추가
- 기존 데이터 접근 로직도 모두 `user_id` 기준으로 수정합니다.
- 배포 전 Supabase에서 `multiusers-ref.sql` 실행을 완료해야 합니다.

# 환경변수/시크릿 로딩 방식 (고정)
- 로컬 실행 시: `.env`를 로드해서 Supabase/OpenAI 키를 읽습니다.
- Streamlit Cloud 배포 시: `st.secrets`에서 키를 읽습니다.
- 권장 우선순위:
  1) `st.secrets`에 값이 있으면 사용
  2) 없으면 `.env`/`os.getenv` 사용
- 지원 키:
  - `SUPABASE_URL`
  - `SUPABASE_ANON_KEY`
  - `OPENAI_API_KEY`
- 키 누락 시 예외로 즉시 종료하지 말고, 사용자에게 누락된 키를 안내합니다.
- `.env` 경로는 `fintech/.env` (실행 위치와 무관하게 절대경로 기준으로 `load_dotenv` 사용)
- `get_config_value(key)` 헬퍼로 secrets/env 통합 로딩을 구현합니다.

# 로깅 (Streamlit Cloud 대응 — 필수)
- Streamlit Cloud(`/mount/src/...`)는 앱 소스 디렉터리에 **쓰기 권한이 없습니다**.
- 시작 시 `LOG_DIR.mkdir()`을 무조건 호출하면 `PermissionError`로 앱이 즉시 종료됩니다.
- 반드시 아래 방식으로 로깅을 구현합니다.
  1) 쓰기 가능한 경로를 순서대로 시도: `fintech/logs` → OS 임시 폴더(`tempfile.gettempdir()/multiusers_logs`)
  2) 각 경로에 대해 mkdir + probe 파일 쓰기 테스트 후 사용
  3) 모두 실패 시 **파일 로깅 없이 콘솔(StreamHandler)만** 사용
  4) `PermissionError`, `OSError`는 잡아서 앱 시작을 막지 않음
- 로그 레벨: WARNING 이상 (httpx, openai 등 noisy logger도 WARNING으로 억제)

# Streamlit Cloud 배포 (체크리스트)
배포 시 아래 순서를 따릅니다.

- Supabase SQL Editor에서 `multiusers-ref.sql` 전체 실행
- `user`, `chat_sessions`, `chat_messages`, `vector_documents` 테이블 및 RPC 생성 확인

## 2. GitHub 업로드
- `multiusers.py`, `requirements.txt`, `multiusers-ref.sql` 포함
- `logo.png`는 저장소 루트(`fintech/logo.png`)에 두면 표시됨 (없어도 앱은 동작)

## 3. Streamlit Cloud 앱 설정
- Main file path: 배포 구조에 맞게 지정 (예: `9.MultiService/code/multiusers.py` 또는 루트에 올린 경우 `multiusers.py`)
- Python 3.11 권장

## 4. Secrets 설정 (Manage app → Settings → Secrets)
```toml
SUPABASE_URL = "https://xxxx.supabase.co"
SUPABASE_ANON_KEY = "eyJ..."
OPENAI_API_KEY = "sk-..."
```

## 5. 배포 후 확인
- 로그인/회원가입 화면이 먼저 표시되는지
- 회원가입 → 자동 로그인 → 대시보드 진입
- PDF 업로드·RAG 답변·세션 저장/로드/삭제
- `PermissionError`(로그 폴더) 없이 앱이 정상 기동

# 기술 사항
- LLM 모델: `gpt-4o-mini` (고정)
- Embedding: OpenAI (`text-embedding-ada-002` 계열, vector(1536))
- PDF 처리: PyPDFLoader + RecursiveCharacterTextSplitter (chunk 500, overlap 100)
- 벡터 검색: Supabase RPC `match_vector_documents` 직접 호출 (세션·user_id 필터)
- `requirements.txt` 필수 패키지:
  - streamlit, python-dotenv, langchain-community, langchain-core
  - langchain-openai, langchain-text-splitters, supabase, openai, pypdf
- 로컬 실행:
  ```powershell
  cd 9.MultiService\code
  pip install -r requirements.txt
  streamlit run multiusers.py
  ```

# 오류 방지 (이전 이슈 반영)
## 1. Streamlit Cloud PermissionError (로그 폴더)
- 원인: `fintech/logs` mkdir 시 읽기 전용 파일시스템
- 해결: 위 "로깅 (Streamlit Cloud 대응)" 섹션대로 fallback 처리

## 2. vector_documents file_name NULL
- PDF 저장 시 `file_name`을 각 청크 row에 명시적으로 포함
- SupabaseVectorStore 자동 매핑에 의존하지 말고 직접 insert

## 3. RAG 검색 필터
- SupabaseVectorStore retriever filter 대신 RPC `match_vector_documents` 직접 호출
- `filter_session_id`, `filter_user_id` 파라미터로 이중 필터링
