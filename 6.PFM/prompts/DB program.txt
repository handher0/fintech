# 역할
너는 사용자의 마이데이터 금융 정보를 정밀 분석하는 핀테크 데이터 분석가이자,
Python 기반 데이터 시각화 및 Streamlit 분석 시스템을 구축하는 전문 웹 솔루션 엔지니어다.
통합 완료된 가계부 데이터베이스를 기반으로, 사용자의 소비 성향을 탐색, 비교, 분석, 진단할 수 있는 "개인 맞춤형 자산 관리 가계부 분석 대시보드 프로그램"을 작성하라.

# 프로젝트 구조 및 경로 (고정)
프로젝트 루트: ./6.PFM/
- data/user_card_history_merged.csv  → 분석 대상인 1월~5월 통합 마스터 가계부 데이터
- code/app_dashboard.py              → 지금 네가 새로 작성해야 하는 Streamlit 분석 프로그램 위치

# 데이터베이스 정보 및 구조 (우리 가계부 데이터셋)
- 데이터 소스: 로컬 통합 CSV (`./6.PFM/data/user_card_history_merged.csv`) 또는 동일 구조의 SQLite 연동 데이터
- 컬럼 구조:
  * month (text, YYYY-MM 형식, 2026-01 ~ 2026-05)
  * Approved_Num (text, 8자리 카드 승인번호)
  * Approved_DateTime (text, YYYY-MM-DD HH:MM:SS 결제일시)
  * Card_Name (text, 사용 카드 종류: '신한 딥드림', '네이버페이 현대카드', '토스뱅크 체크카드')
  * Store_Name (text, 결제 가맹점명: '배달의민족', '스타벅스', '카카오T(택시)' 등)
  * Category (text, 소비 대분류: '식비/배달', '카페/간식', '교통/차량', '쇼핑/뷰티', '문화/생활')
  * Amount_KRW (integer/float, 원화 결제 금액)

# 프로그램 목표
Streamlit 기반 "개인 맞춤형 자산 관리 가계부 분석 대시보드 시스템" 구축
이 프로그램은 단순한 표 나열이 아니라, 핀테크 현업에서 사용하는 수준의 다음 분석 기능이 유기적으로 연결된 전문 PFM 도구여야 한다.
- 소비 데이터 탐색 (EDA) 및 동적 필터링
- 월별 지출액 및 결제 건수 동향 분석
- 카테고리별 / 신용카드별 소비 효율 비교
- 야간 배달 및 출퇴근 택시 등 과소비 변동성 진단
- 고정 지출 및 핵심 이슈 가맹점 탐색

# 필수 UI 및 기능 요구사항

## 1. 데이터 로드 및 결측치 방어
- `pandas`를 활용해 마스터 데이터를 안전하게 로드하되, 꼬리행(`Total`)이나 금액이 0원 이하, 혹은 Null/NaN인 유효하지 않은 데이터는 계산에서 원천 배제하는 방어 코드를 포함하라.
- 로드 성공 시 총 누적 데이터 건수를 화면에 작게 표시하고, 예외 발생 시 친절한 안내 메시지를 출력하라.

## 2. 사이드바 인터랙티브 필터 시스템
사용자가 마이데이터를 다각도로 쪼개어 분석할 수 있도록 사이드바에 실시간 필터 조작 패널을 명확히 구현하라:
- 조회 기간 (month): 멀티 선택 컴포넌트 (`st.multiselect`)
- 소비 분류 (Category): 멀티 선택 컴포넌트
- 보유 카드 (Card_Name): 멀티 선택 컴포넌트
- 자주 가는 가맹점 (Store_Name): 검색 텍스트 및 선택 컴포넌트
- 결제 금액 범위 (Amount_KRW): 최소~최대 지출액 제어 슬라이더 (`st.slider`)
※ 사용자가 필터를 변경하는 즉시 중앙 메인 화면의 모든 KPI 지표와 시각화 차트가 실시간 갱신(Re-rendering)되도록 연동하라.

## 3. 데이터 탐색 및 요약 통계 영역 (EDA)
- 필터링된 실시간 가계부 DataFrame 테이블을 가독성 있게 노출하라.
- 현재 선택된 조건의 총 지출 건수, 고유 가맹점 수, 분석 대상 카테고리 개수를 서두에 요약 출력하라.

## 4. 토스형 금융 핵심 KPI 영역
메인 화면 상단에 `st.metric`을 가로 레이아웃으로 배치하여 다음의 핵심 재무 지표를 직관적인 카드로 노출하라:
- 필터 조건 내 [총 소비 금액] (천단위 콤마 및 '원' 표시)
- 건당 [평균 결제 금액]
- 지출 비중이 가장 높은 [최다 소비 카테고리] 및 해당 금액
- 단일 결제 기준 [최고 지출 금액] 및 결제처
- 표준편차가 가장 커서 지출 통제가 필요한 [최대 변동성 카테고리]

## 5. 시각화 시스템 (분석 목적별 섹션 구조)
모든 차트는 인터랙티브 기능(확대, 툴팁, 범례 클릭)이 지원되는 Plotly 또는 Altair 라이브러리를 활용하여 "목적별 섹션"으로 나누어 빌드하라.

### (1) 월별 지출 동향 및 추이 분석
- 시각화 유형: 꺾은선 그래프 (Line Chart) 또는 누적 영역 차트 (Area Chart)
- 축 설정: x = month, y = Amount_KRW의 합계
- 그룹화(Color): Category 또는 Card_Name을 유저가 선택하여 교차 비교할 수 있도록 양방향 컴포넌트 연동
- 분석 목적: 상반기 소비가 우상향하는지, 특정 월에 지출이 폭발했는지 시계열 흐름 파악

### (2) 카드별 / 카테고리별 소비 효율 비교
- 시각화 유형: 그룹형 또는 누적 막대 그래프 (Bar Chart)
- 분석 목적: 어떤 카드를 쓸 때 지출이 가장 컸는지, 카테고리별 지출 규모(유통 채널별 금액 차이) 비교

### (3) 소비 이상치 및 지출 변동성 파악
- 시각화 유형: 박스 플롯 (Box Plot)
- 축 설정: x = Category, y = Amount_KRW
- 분석 목적: 사용자의 일상적인 소비 금액대 분포를 확인하고, 야간 4~5만 원대 배달 음식이나 고액 쇼핑 같은 '이상 과소비 지출(Outlier)'을 시각적으로 즉각 탐색

### (4) 시간대별 / 요일별 소비 구조 히트맵
- 시각화 유형: 밀도 히트맵 (Density Heatmap) 또는 피벗 테이블 기반 컬러 매핑
- 축 설정: x = 결제 요일(또는 월), y = Category, 명도/색상 = 평균 지출액
- 분석 목적: 계절성 소비, 주말 집중 소비, 구조적 과소비 구간(패턴 A, B)을 시각적으로 탐지

### (5) 과소비 주범 Top-N 가맹점 추적
- 시각화 유형: 가로 막대 그래프 (Horizontal Bar Chart)
- 분석 목적: 사용자의 지갑을 가장 위협하는 '지출 상위 Top 10 가맹점(예: 배달의민족, 카카오T)'을 순위별로 랭킹화

## 6. 마이데이터 기반 소비 동향 자동 요약 리포트 (최하단)
선택된 필터 조건과 통계 수치 연산 결과를 조합하여, 실제 자산 관리 전문가가 작성한 듯한 잔소리 텍스트를 규칙 기반(Rule-based) 문장 생성 템플릿으로 출력하라.
- [예시 형태]: "2026년 1월부터 5월까지 사용자의 누적 지출액은 총 1,798,607원이며, 이 중 가장 큰 비중을 차지하는 과소비 주범은 '식비/배달(995,635원)'입니다. 특히 배달의민족과 쿠팡이츠에서의 야간 지출 변동성이 매우 크게 나타났습니다. 지출 통제를 위해 다음 달 배달 음식 5만 원 줄이기 챌린지에 도전할 것을 권장합니다."

# 코드 구조 및 품질 요구사항
- 기능별 함수 단위 설계 및 `main()` 실행 구조 준수:
  * `load_data()`: 데이터 로드 및 전처리
  * `apply_filters()`: 사이드바 조건 매칭
  * `calculate_kpi()`: 통계 및 metric 연산
  * `draw_trend_charts()`, `draw_compare_charts()`, `draw_distribution_charts()`, `draw_heatmap()`, `draw_top_items()`: 차트 드로잉 유틸리티
- 코드 상단에 프로그램의 구동 환경 및 데이터 정보 주석 기술.
- 사용자가 필터를 과도하게 적용하여 데이터가 0건이 되었을 때 화면이 뻗지 않도록 `if filtered_df.empty:` 예외 처리를 수행하여 화면에 "조건에 맞는 가계부 내역이 없습니다."를 예쁘게 띄울 것.
