# 역할 및 목표
너는 개인 자산 관리(PFM) 및 마이데이터 금융 서비스를 개발하는 수석 데이터 엔지니어다.
우리가 가진 요구사항에 맞춰 월별 엑셀 파일들을 하나로 병합하는 **'파이썬 마이그레이션 프로그램 소스코드(code/merge.py)'를 작성**하는 것이 네 임무다.

★ [경고 및 절대 제약사항] ★
- 너는 절대 이 자리에서 데이터를 실제로 읽거나, 병합을 수행하거나, csv 파일을 직접 생성해서는 안 된다.
- 너의 유일한 아웃풋은 사용자가 지정된 경로(`./6.PFM/code/merge.py`)에 저장하고 실행할 수 있는 **'완전하고 실행 가능한 Python 소스코드'**여야 한다.

# 프로젝트 구조 및 경로 (고정)
프로젝트 루트: ./6.PFM/
- data/         → 1월~5월의 입력 Excel 파일들이 위치한 곳이자, 최종 CSV가 저장될 위치
- code/merge.py → 지금 네가 새로 작성해야 하는 파이썬 머지 소스코드 위치

# 입력 파일 정보 (작성할 코드가 처리해야 할 대상)
- 파일 대상 기간: 2026년 1월 ~ 5월 (총 5개 파일)
- 파일명 패턴: user_card_history_2026_X.xlsx (X는 01부터 05까지의 숫자)
- 시트 구조: 각 파일의 첫 번째 활성화된 시트(Active Sheet)를 읽어서 처리하며, 각 시트명은 `2026년 X월 내역` 형태임.

# 입력 파일 내부 구조 (코드가 파싱해야 할 규칙)
- 행 1: 헤더 영역 (Approved_Num, Approved_DateTime, Card_Name, Store_Name, Category, Amount_KRW)
- 행 2 ~ 끝: 실제 결제 데이터 행
- ★ [주의사항]: 각 엑셀 파일의 최하단 행에는 `=SUM(F2:F...)` 구조의 'Total' 합계 행이 포함되어 있음. 따라서 데이터를 읽을 때 첫 번째 열(A열)의 값이 "Total"인 행은 실제 데이터가 아니므로 반드시 필터링하여 제외(Skip)해야 함.

# 핵심 데이터 변환 및 정제 로직 (코드에 녹여낼 내용)
1. 월(month) 식별 정보 추출 및 추가
   - 각 파일에서 데이터를 추출할 때, 해당 데이터가 몇 월 데이터인지 구분할 수 있도록 최종 CSV에 `month`라는 새로운 컬럼을 맨 첫 번째 열에 추가해야 함.
   - 파싱 포맷: 파일명 또는 시트명에서 월을 추출하여 YYYY-MM 형태로 저장 (예: user_card_history_2026_01.xlsx에서 데이터를 읽을 때 month 값은 2026-01로 고정 적용).

2. 작업 순서 보장
   - 파일 처리 시 파일명 오름차순(1월 파일부터 5월 파일 순서)으로 정렬하여 유기적인 시계열 순서대로 결합되도록 하라.

3. 데이터 파싱 및 결측치 방어 규칙
   - openpyxl 라이브러리를 사용할 때 반드시 data_only=True 옵션을 주어 하단 Total 행 등의 수식이 아닌 계산된 결과값 셀을 안전하게 읽어올 수 있도록 하라.
   - 데이터 행을 순회하는 중, 필수 값인 Approved_Num이나 Amount_KRW 셀이 완전히 비어있거나 None인 행이 있다면 빈 데이터로 간주하고 레코드를 생성하지 않고 건너뛴다.

# 코드 품질 및 패키지 의존성 요구사항 (중요)
- 기능별 함수 단위 구조 설계 및 `if __name__ == "__main__": main()` 실행문 필수 포함.
- glob 또는 os.listdir를 사용한 파일 로드 단계부터 파일 병합, 하단 Total 행 필터링, 최종 CSV 파일 쓰기 단계까지 터미널에 상세한 진행 로그(print)를 출력할 것.
- 의존성(openpyxl) 에러 방지: 코드 최상단에서 openpyxl을 import 할 때 try-except 블록을 사용하여 ModuleNotFoundError가 발생하면 터미널에 "[오류] openpyxl 라이브러리가 설치되어 있지 않습니다. 터미널에서 'pip install openpyxl'을 실행한 후 다시 시도해주세요."라는 메시지를 출력하고 프로그램이 안전하게 종료(sys.exit(1))되도록 로직을 추가할 것.
- 엑셀 파싱 중 예상치 못한 에러로 인해 프로그램이 멈추지 않도록 try-except 예외 처리를 철저히 적용할 것.

# 최종 출력 CSV 스펙 (코드가 생성해낼 결과물의 스펙)
- 파일 저장 경로: ./6.PFM/data/user_card_history_merged.csv
- 인코딩 방식: Excel에서 열었을 때 한글 가맹점명과 카테고리가 깨지는 것을 원천 차단하기 위해 반드시 UTF-8 BOM (utf-8-sig) 형식으로 지정하여 저장할 것.

  month, Approved_Num, Approved_DateTime, Card_Name, Store_Name, Category, Amount_KRW
