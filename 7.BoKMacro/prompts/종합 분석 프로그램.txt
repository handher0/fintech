# 역할
너는 Python 기반 금융 데이터 사이언티스트이자, 한국은행 Open API(ECOS)와 LLM 에이전트, 클라우드 자동 보고서 파이프라인을 결합하여 매크로 리서치 솔루션을 제공하는 수석 금융 엔지니어다.
1단계(수집 및 챗봇)와 2단계(상관관계 분석)의 모든 기능을 하나의 완성형 인터페이스로 통합하고, 사용자가 요청할 때만 OpenAI 최신 API 및 정적 그래프가 포함된 고품질 자산 진단 리포트를 추출하는 "Streamlit 기반 거시경제 종합 분석 및 보고서 자동 생성 시스템(macro_analysis_system.py)" 소스코드를 작성하라.

# 핵심 구조 및 두 가지 모드의 명확한 분리
1. [기본 모드 - 실시간 모니터링 & AI 에이전트] (빠르고 가볍게 구동)
   - 한국은행 API 기반 실시간 환율, 금리, 물가 데이터 연동 및 가로형 KPI 대시보드 표기.
   - 자연어 질문을 해석하여 Plotly 차트(시계열, 히트맵, 산점도)를 채팅창 내에 즉각 피드백하는 금융 SQL/Data 챗봇 가동.
2. [리포트 모드 - 거시경제 동향 보고서 생성] (사용자가 버튼을 클릭했을 때만 무겁고 정밀하게 구동)
   - 통계 연산 결과와 OpenAI 실시간 글로벌 트렌드 웹 검색 데이터를 병합하여 python-docx 기반의 마스터 한글 보고서(.docx) 파일 생성 및 다운로드 기능 제공.

# 핵심 데이터 파이프라인 및 변환 가이드라인
- **데이터 범위**: 통계 연산 및 트렌드 분석을 위해 **2025년 1월 ~ 현재(2026년)** 데이터를 풀링(Pooling)하되, 차트와 챗봇 인터페이스의 주 시각화 경계는 2026년 이후 데이터에 초점을 맞추라.
- **데이터 동기화**: 일별 환율 데이터와 월별 금리/물가 데이터를 `month(YYYY-MM)` 기준으로 inner join 결합하여 피어슨 상관계수 행렬을 계산하는 2단계 전처리 모듈을 완벽하게 내장하라.

# 필수 UI 및 기능 요구사항

1. 상단 마이데이터 금융 KPI 및 탭(Tab) 구조
   - 상단 레이아웃에 최신 환율, 기준금리, CPI 전년동월비 수치를 가로형 메트릭 카드로 브리핑하라.
   - 중앙 영역은 탭 구조를 채택하여 레이아웃을 정돈하라:
     * [Tab 1: AI 매크로 에이전트] -> 실시간 대화창 및 동적 차트 출력 영역
     * [Tab 2: 통계적 자산 상관관계] -> 히트맵, 복합 대조 차트, 회귀 추세선 산점도 모아보기

2. 대화형 분석 영역 (AI 경제 챗봇 UI)
   - `st.chat_message`와 `st.chat_input` 기반의 채팅 창을 구성하라. (안내: "거시경제 지표 및 자산 영향에 대해 질문하세요.")
   - 답변 양식 제한: 줄바꿈과 순수 텍스트 위주의 문단, 그리고 불렛 리스트(-)만 허용하라. (서식 깨짐 및 docx 마이그레이션 오류 방지를 위해 `#`, `**`, `` ` `` 등 마크다운 태그 사용 금지 — `_strip_markdown` 후처리 적용).

3. 리포트 생성 및 다운로드 엔진 (리포트 모드 전용)
   - 챗봇 답변 하단 또는 사이드바 하단에 "종합 자산 보고서(DOCX) 생성" 버튼을 명확히 노출하라.
   - **[버튼 작동 시 워크플로우]**:
     ① 종합 통계 수치 및 상관관계 분석 데이터프레임 스냅샷 확보.
     ② **[Matplotlib 정적 차트 백업]**: 무거운 외부 의존성인 kaleido를 절대 사용하지 말고, Matplotlib/Seaborn으로 시계열 추이 그래프와 상관관계 히트맵을 정적 PNG 이미지 파일로 `output/charts/`에 저장하라.
     ③ **[OpenAI 웹 검색 연동]**: 사이드바 검색 옵션이 켜져 있을 경우, `client.responses.create` 또는 openai 최신 SDK를 활용하여 "2026년 현재 미국 연준(Fed) 금리 방향 및 글로벌 인플레이션 원인"에 대한 최신 검색 텍스트 데이터를 확보하라.
     ④ **[python-docx 문서 빌드]**: 표지, 요약, 데이터 분석 통계표, 백업된 PNG 이미지 삽입(`doc.add_picture()`), 웹 검색 결합 원인 분석, 종합 자산운용 결론 순으로 구조화된 문서를 작성하여 `output/reports/`에 저장하고 `st.download_button`을 활성화하라.
   - **[DOCX 한글 서식 방어 규칙 (필수)]**:
     * 날짜·기간 문자열은 `strftime`을 쓰지 말고 수동 포맷(`2026년06월23일`)으로 작성하라.
     * 모든 Run에 `맑은 고딕` 폰트를 eastAsia/ascii/hAnsi에 명시하고, 문단 `autoSpaceDE`·`autoSpaceDN`을 0으로 비활성화하라.
     * 상관계수는 `DataFrame.to_string()` 텍스트 덤프 금지 — 반드시 Table Grid 표로 렌더링하라.
     * 차트 위에는 내부 파일명(`exchange_rate_타임스탬프`) 대신 한글 캡션(예: `그림 1. 원/달러 환율 추이`)을 표기하라.
     * 표지는 단일 셀 테이블 가운데 정렬로 작성해 제목 글자 간격 깨짐을 방지하라.
     * 보고서 생성 후 `st.session_state`의 이전 DOCX 캐시를 무효화하고, 새 파일명으로 다시 다운로드되게 하라.

4. 사이드바 제어 패널
   - LLM 모델 선택 컴포넌트, OpenAI 실시간 글로벌 금융 트렌드 웹 검색(web_search) 온오프 체크박스 구현.
   - 대화 내역 및 캐시를 완전히 리셋하는 '대화 초기화(Reset Chat)' 버튼 구현.

5. 예외 처리 방어 규칙
   - 한국은행 API 통신 지연이나 OpenAI 할당량 부족(401, Quota) 에러 발생 시 시스템 전체가 Crash 나지 않도록 철저히 `try-except`로 예외화하여 안내 창을 노출하라.
