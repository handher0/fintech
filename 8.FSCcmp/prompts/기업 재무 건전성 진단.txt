# 역할
너는 Python 기반 금융 데이터 엔지니어이자, 기업의 재무제표(재무상태표, 손익계산서) 데이터를 정밀 분석하여 기업의 신용 위험과 내재 가치를 평가하는 퀀트 투자 솔루션 개발자다.
`금융위원회_기업재무정보` API 및 주식시세 데이터를 연동하여 주요 상장 기업들의 재무 비율을 실시간 연산하고, 사용자가 설정한 부채비율 및 영업이익률 마지노선에 따라 우량주를 필터링하는 알고리즘을 구현하라. 또한 듀퐁 분석(DuPont Analysis) 기법을 시각화하고, AI 신용평가원 비서 기능을 내장한 "Streamlit 기반 AI 기업 재무 건전성 진단 플랫폼(financial_health_agent.py)" 소스코드를 작성하라.

# 핵심 구현 목표 (★ 절대 인프라/DB 생성 코드를 짜지 말 것)
너는 오직 기업재무정보 API 호출 및 연간/분기별 재무 항목(자산총계, 부채총계, 자본총계, 매출액, 영업이익, 당기순이익) 파싱, 재무 비율(부채비율, 영업이익률, ROE) 연산 알고리즘, 듀퐁 분석 차트 구현, `st.chat_message` 기반의 AI 신용평가원 인터페이스 화면 구현에만 집중하라.

# 환경 변수 및 .env 파일 로드 방어 규칙
1. 파이썬 스크립트 실행 위치(Cwd)에 영향을 받지 않고 프로젝트 루트인 `fintech/.env`를 정확히 조준할 수 있도록 경로 인식 코드를 설계하라.
2. 현재 파일(`fintech/8.FSCcmp/code/financial_health_agent.py`)의 절대 경로 기준으로 상위 폴더 3개를 거슬러 올라가 상위 루트인 `fintech` 디렉토리를 찾아내고, 해당 경로에 위치한 `.env` 파일을 명시적으로 로드하라.
   - 구조: `Path(__file__).resolve().parent.parent.parent / '.env'`
3. `load_dotenv(dotenv_path=env_path, override=True)`를 수행한 뒤, `utf-8-sig` 인코딩 방식으로 `KEY=VALUE` 형식을 병합 검증하여 `PUBLIC_DATA_API_KEY` 및 `OPENAI_API_KEY`를 안정적으로 바인딩하라. Key 누락 시 `st.error`로 차단하라.

# 금융위원회 Open API 호출 및 재무 데이터 전처리 규칙
- **기업재무정보 엔드포인트 URL**:
  `http://apis.data.go.kr/1160100/service/GetFinaStatInfoService/getMsemasBsisStat` (또는 대표 재무제표 조회 서비스 주소 활용)
- URL 주소 자체에 `?serviceKey={PUBLIC_DATA_API_KEY}`를 직접 f-string 바인딩하여 호출 파라미터 에러를 원천 방어하라.
- 데이터 조회 구간: **최근 3개년 시계열 재무 데이터** (2024년 ~ 현재 2026년 기준 시점까지의 연간/분기 보고서 데이터 대상)
- **[재무 데이터 정제 방어 코드]**: 기업재무정보 API는 계정과목명(`account_nm` 등)이나 수치 데이터의 결측치(NaN) 변동성이 크다. 주요 타깃 기업들(예: 삼성전자, SK하이닉스, 현대차 등)의 자산, 부채, 자본, 매출액, 영업이익, 당기순이익 항목을 정확히 매핑하여 딕셔너리 및 데이터프레임으로 변환하고, 누락된 값은 직전 분기/연도 데이터로 보완(Fillna)하거나 예외 처리하여 크래시를 방어하라.
- API 데이터 요청 및 연산 로직은 `@st.cache_data(ttl=3600)`으로 캐싱하라.

# 필수 UI 및 기능 요구사항

1. 상단 디자인 및 브랜드 로고 영역
   - [상단 헤더 그라데이션 디자인 커스텀 (CSS 인젝션)]:
     * `st.markdown`의 `unsafe_allow_html=True`를 활용하여 메인 타이틀 직전에 왼쪽에서 오른쪽으로 부드럽게 이어지는 '레드-블루 그라데이션(linear-gradient(to right, #FF4B4B, #1C65E3))' 효과가 적용된 세련된 헤더 바(고정 높이 8px)를 구현하라.
   - [브랜드 로고 파일 노출]:
     * 프로젝트 루트 폴더에 존재하는 로고 파일(`./fintech/logo.png`)을 절대 경로 계산식으로 찾아 파일 유효성을 검증한 후, 사이드바 맨 상단 영역에 `st.sidebar.image`로 컴팩트하게 노출하라.
   - 타이틀: "🏢 AI 기반 기업 재무 건전성 진단 분석 에이전트"

2. 사이드바 안정성/수익성 필터 툴 (Screening)
   - 사이드바에 유저가 기업의 재무적 마지노선을 설정할 수 있는 두 개의 입력 컴포넌트를 배치하라:
     ① **부채비율 한계치 설정 슬라이더** (`st.slider`, 범위: 0% ~ 300%, 기본값: 150% 이하)
     ② **영업이익률 최소 기준 설정 슬라이더** (`st.slider`, 범위: -10% ~ 50%, 기본값: 5% 이상)
   - 사용자가 슬라이더를 조정하면, 수집된 기업 데이터 중 **[부채비율 = (부채총계/자본총계)*100]** 및 **[영업이익률 = (영업이익/매출액)*100]** 조건을 동시에 만족하는 우량 기업 종목 리스트만 메인 화면 상단 테이블(`st.dataframe`)에 실시간 필터링하여 바인딩하라.

3. 듀퐁 분석(DuPont Analysis) 시각화 섹션 (중앙 영역)
   - 특정 기업을 선택하면 해당 기업의 **자기자본이익률(ROE)** 성과를 심층 해부하는 듀퐁 분석 인포그래픽 차트를 Plotly로 구현하라.
   - **[듀퐁 분석 분해 산식]**: $ROE = 순이익률(당기순이익/매출액) \times 자산회전율(매출액/자산총계) \times 재무레버리지(자산총계/자본총계)$
   - 시각화: Plotly의 `bar` 차트 혹은 트리맵(Treemap), 혹은 세 개의 지표가 멀티 플롯 형태로 결합된 레이아웃을 사용하여, 최근 3개년 동안 해당 기업의 ROE 상승/하락을 견인한 핵심 재무 요인이 무엇인지 직관적으로 보여주는 복합 차트를 렌더링하라.

4. 하단 대화형 AI 신용평가원 비서 영역 (AI 에이전트 UI)
   - `st.chat_message`와 `st.chat_input` 창으로 구성된 기업 신용 평가 챗봇 영역을 빌드하라. (입력창 안내: "이 기업의 3개년 재무 추이를 볼 때 고금리 시대를 버틸 수 있을까?")
   - 유저가 특정 기업에 대해 질문하면, 해당 기업의 3개년 부채비율, 이자보상배율, 유동비율, 듀퐁 분해 수치 데이터프레임 스냅샷을 LLM(OpenAI) 컨텍스트로 주입하여 정량적 재무 비율을 근거로 투자 위험 조언을 하도록 로직을 작성하라.
   - 시스템 프롬프트 페르소나: "너는 공인회계사(CPA)이자 기업의 부도 위험과 재무 건전성을 냉철하게 심사하는 금융기관 수석 신용평가원 비서다."
   - 답변 서식 깨짐 방지를 위해 `#`, `**`, `` ` `` 등 마크다운 태그 사용을 전면 금지하고, 일반 문단 텍스트와 불렛 리스트(-)만 허용하는 후처리 (`_strip_markdown`)를 적용하라.

5. 제어 패널
   - LLM 모델 선택 컴포넌트 및 '대화 초기화(Reset)' 버튼 구현.
