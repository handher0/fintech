# 역할
너는 Python 기반 금융 데이터 사이언티스트이자, 다차원 자산 데이터를 계량화(Quant)하여 필터링하고 종목 간 비교 분석을 수행하는 핀테크 솔루션 개발자다.
`금융위원회_주식시세정보` API를 활용하여 시장 내 주요 복수 종목(또는 업종별 대표 종목)의 다차원 금융 데이터를 실시간 풀링(Pooling)하고, 사용자가 설정한 복합 조건에 따라 데이터를 정밀 필터링(Screening)하는 알고리즘을 구현하라. 또한, 선택 종목 간의 성과를 다차원 차트로 대조하고 자연어로 퀀트 진단을 수행하는 "Streamlit 기반 다차원 금융 데이터 비교·필터링 에이전트 플랫폼(financial_screening.py)" 소스코드를 작성하라.

# 핵심 구현 목표 (★ 데이터 다차원 가공 및 융합에 집중)
너는 오직 복수 종목 API 호출 및 단일 마스터 데이터프레임 통합, 파생 지표(수익률, 변동성, 거래대금) 연산 알고리즘, 조건별 동적 필터링 매커니즘, Streamlit 다차원 비교 UI 렌더링, `st.chat_message` 기반의 AI 퀀트 에이전트 인터페이스 로직 구현에만 집중하라.

# 환경 변수 및 .env 파일 로드 방어 규칙 (상위 경로 고정)
1. 파이썬 스크립트 실행 위치(Cwd)에 영향을 받지 않고 프로젝트 루트인 `fintech/.env`를 정확히 조준할 수 있도록 경로 인식 코드를 설계하라.
2. 현재 파일(`fintech/8.FSCcmp/code/financial_screening.py`)의 절대 경로 기준으로 상위 폴더 3개를 거슬러 올라가 상위 루트인 `fintech` 디렉토리를 찾아내고, 해당 경로에 위치한 `.env` 파일을 명시적으로 로드하라.
   - 구조: `Path(__file__).resolve().parent.parent.parent / '.env'`
3. `load_dotenv(dotenv_path=env_path, override=True)`를 수행한 뒤, `utf-8-sig` 인코딩 방식으로 `KEY=VALUE` 형식을 병합 검증하여 `PUBLIC_DATA_API_KEY` 및 `OPENAI_API_KEY`를 안정적으로 바인딩하라. Key 누락 시 `st.error`로 차단하라.

# 금융위원회 Open API 호출 및 다차원 데이터셋 빌드 규칙
- 주식시세정보 엔드포인트 URL 주소 자체에 `?serviceKey={PUBLIC_DATA_API_KEY}`를 직접 f-string 바인딩하여 호출 파라미터 꼬임을 원천 방어하라.
- 데이터 조회 구간: **2026년 1월 1일 ~ 현재**
- **[다차원 마스터 데이터프레임 구축 로직]**:
  * 대시보드 구동 시 주요 대표 종목(예: 삼성전자, SK하이닉스, 현대차, 네이버 등 최소 4~5개 종목)의 시계열 데이터를 API로 각각 호출하라.
  * 호출된 개별 데이터프레임을 하나의 마스터 데이터프레임으로 병합(Concat 또는 Merge)하되, 각 행이 `[날짜(basDt), 종목명(itmsNm), 종가(clpr), 거래량(trqu), 거래대금(trPrc), 등락률(fltRt)]`을 가지도록 정형화하라.
  * Pandas를 활용하여 종목별 **'최근 20일 누적 수익률'**, **'최근 20일 주가 변동성(표준편차)'**, **'평균 거래대금'**을 다차원 파생 변수로 자동 연산하는 알고리즘을 내장하라.
- API 데이터 요청 및 연산 로직은 `@st.cache_data(ttl=1800)`으로 캐싱하라.

# 필수 UI 및 기능 요구사항

1. 상단 디자인 및 브랜드 로고/KPI 영역
   - [상단 헤더 그라데이션 디자인 커스텀 (CSS 인젝션)]:
     * `st.markdown`의 `unsafe_allow_html=True`를 활용하여 메인 타이틀 직전에 왼쪽에서 오른쪽으로 부드럽게 이어지는 '레드-블루 그라데이션(linear-gradient(to right, #FF4B4B, #1C65E3))' 효과가 적용된 세련된 헤더 바(고정 높이 8px)를 구현하라.
   - [브랜드 로고 파일 노출]:
     * 프로젝트 루트 폴더에 존재하는 로고 파일(`./fintech/logo.png`)을 절대 경로 계산식으로 찾아 파일 유효성을 검증한 후, 사이드바 맨 상단 영역에 `st.sidebar.image`로 알맞은 너비로 컴팩트하게 노출하라.
   - 타이틀: "📊 다차원 금융 데이터 조건별 스크리닝 및 비교 분석 플랫폼"

2. 사이드바 다차원 조건 필터링(Screening) 알고리즘 UI
   - 사이드바에 사용자가 마스터 데이터셋을 필터링할 수 있는 **계량적 조건 슬라이더 및 체크박스 컴포넌트**를 배치하라:
     ① 최소 등락률/수익률 조건 슬라이더
     ② 최소 거래대금 마지노선 설정 슬라이더
     ③ 주가 변동성 범위(임계치) 제한 필터
   - 사용자가 사이드바의 필터 조건을 조정하면, 마스터 데이터프레임에 Pandas 불리언 인덱싱 알고리즘이 실시간 적용되어 조건에 부합하는 종목과 일자만 솎아낸 **'조건별 스크리닝 결과 테이블(`st.dataframe`)'**을 메인 화면 중앙에 실시간으로 노출하라.

3. 종목 간 다차원 비교 시각화 섹션 (메인 화면 중앙)
   - 스크리닝된 데이터를 기반으로 두 개 이상의 종목을 선택하여 성능을 대조할 수 있는 Plotly 인터랙티브 차트 2종을 렌더링하라.
   - **차트 1 (누적 수익률 비교 선형 차트)**: 선택한 종목들의 2026년 이후 누적 수익률 추이를 한 차트에 오버랩하여 비교하는 꺾은선 그래프 (X축: DATE, Y축: 수익률 %).
   - **차트 2 (위험 대비 수익률 산점도 차트)**: X축을 '변동성(위험)', Y축을 '누적 수익률(성과)'로 설정하여, 현재 어떤 종목이 가장 효율적인 투자 위치에 있는지 한눈에 보여주는 버블/산점도(Scatter) 그래프.
   - X축 가독성 규칙: 눈금이 절대 겹치지 않도록 포맷팅(`tickformat="%Y-%m-%d"`, `tickangle=-30`, `nticks=10`)을 철저히 적용하라.

4. 하단 대화형 퀀트 진단 영역 (AI 분석 에이전트 UI)
   - `st.chat_message`와 `st.chat_input` 창으로 구성된 퀀트 분석 챗봇 영역을 빌드하라. (안내: "필터링된 종목들의 투자 매력도와 위험 요인에 대해 질문하세요.")
   - 유저가 "현재 조건에서 가장 안정적인 종목이 무엇인지 퀀트 관점에서 분석해줘"라고 요청하면, 화면에 필터링되어 나타난 요약 통계프레임을 LLM(OpenAI) 컨텍스트로 주입하여 철저히 정량적 데이터 기반으로 브리핑하도록 하라.
   - 시스템 프롬프트 페르소나: "너는 다차원 자산 데이터의 요인(Factor) 분석과 통계적 필터링 결과를 바탕으로 포트폴리오 전략을 조언하는 헤지펀드 퀀트 애널리스트다."
   - 답변 서식 깨짐 방지를 위해 `#`, `**`, `` ` `` 등 마크다운 태그 사용을 전면 금지하고, 일반 문단 텍스트와 불렛 리스트(-)만 허용하는 후처리(`_strip_markdown`)를 적용하라.

5. 제어 패널
   - LLM 모델 선택 컴포넌트 및 '대화 초기화(Reset)' 버튼 구현.
